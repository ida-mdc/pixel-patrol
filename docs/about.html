About Pixel Patrol

Automated dataset
understanding at scale.

Pixel Patrol scans your image collection - from a single image to millions of files - and produces a structured, interactive report. Set up a directory, choose a loader, and get a complete picture of your dataset in a streamlined, reproducible way.

The report captures file metadata, image dimensions, pixel statistics, and quality metrics for every file, with per-dimension breakdowns and group comparisons built in - ready for you to explore and share.

How It Works

How it works

Set up a directory and a few parameters. Pixel Patrol handles the rest.

1
Your dataset

Point Pixel Patrol to your image folder and optional condition subdirectories.

2
Process

Files are read in parallel across all available CPUs. Results are saved to a portable Parquet file.

3
Parquet file

A single portable file holds all metrics, statistics, thumbnails, and metadata. Share it with a colleague and they can open it directly in the viewer.

4
Explore

Browser-based interactive dashboard. Group by condition, filter by any metric, inspect thumbnails, compare distributions.

Key Features

Built for real-world datasets.

Shareable reports

The Parquet file is the report. Anyone can open it in the hosted viewer - no installation, no Python required. Ideal for sharing with collaborators, reviewers, or facility staff.

Group comparison and filtering

Compare experimental conditions side by side with violin plots, histograms, and dimension distributions. Filter by any metric and change groupings interactively - no reprocessing needed.

Per-dimension statistics

For multi-dimensional images (Z, T, C), statistics are computed per slice - not just per file. Detect drift, photobleaching, and channel-specific artifacts at a glance.

Parallel processing and HPC support

Processing runs across all available CPUs using Dask. Connect to an existing HPC cluster to scale across hundreds of workers and process terabyte-scale datasets in minutes.

Any dataset size and format

Images too large to fit in memory are processed in spatial chunks. Container files (LMDB, multi-series OME-TIFF) are handled natively, with each sub-image as an individual entry in the report.

Extensible by design

Add custom loaders, processors, and viewer widgets as standalone Python packages. Entry points are discovered automatically - no fork or viewer rebuild required.
