You've collected hundreds, maybe thousands of image files. Are they all what you think they are? Pixel Patrol scans your dataset and gives you a clear, interactive report before you spend hours running a pipeline on data not suitable for your purpose.

Image collections vary in ways that are easy to miss.

  • Different dimensions, bit depths, or channel counts
  • Metadata from different microscopes or acquisition dates
  • Empty frames, missing time points, or truncated Z-stacks
  • Intensity shifts between batches that affect downstream statistics

Pixel Patrol: the full picture.

Pixel Patrol reads every image in your dataset in parallel and extracts the facts: dimensions, file size, bit depth, channel count, pixel statistics, and any embedded metadata — whatever the format provides.

The results go into a single Parquet file. A fast, DuckDB-powered browser viewer then lets you filter, group, spot outliers, and compare experimental conditions side by side.

The Parquet file is portable. Share it with a colleague and they can drop it straight into the hosted viewer — no Python, no installation.

Four steps from folder to insight

Your dataset

4

Explore

The viewer

A browser-native viewer powered by DuckDB

The Pixel Patrol viewer is a standalone web app built on DuckDB WASM and Plotly. It runs entirely in the browser — no server required for most datasets.

Drag and drop a .parquet file to open it instantly. For large files, pixel-patrol view starts a local HTTP server backed by native DuckDB for significantly faster queries.

The viewer is extensible: plugins are shipped as JavaScript modules and loaded from installed Python packages or any URL. No viewer rebuild needed to add a custom widget.

Get started

Install

Pixel Patrol requires Python 3.11 or higher. We recommend uv for fast, reproducible environments. 

# macOS / Linux:
curl -Ls https://astral.sh/uv/install.sh | sh
# Windows:
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

# Create a virtual environment and install:
uv venv --python 3.12 pixel-patrol-env
source pixel-patrol-env/bin/activate  # Windows: pixel-patrol-env\Scripts\Activate.ps1
uv pip install pixel-patrol

pixel-patrol --help
python -m venv pixel-patrol-env
source pixel-patrol-env/bin/activate  # Windows: pixel-patrol-env\Scripts\Activate.ps1
pip install pixel-patrol

pixel-patrol --help
# Core only — add exactly what you need:
uv pip install pixel-patrol-base

# Optional add-ons:
uv pip install pixel-patrol-loader-bio   # BioIO + Zarr (CZI, ND2, LIF, TIFF, …)
uv pip install pixel-patrol-image        # image quality metrics + extra widgets

CLI commands

Two commands cover the standard workflow; the rest are convenience tools.

Step 1 pixel-patrol process <dir> -o out.parquet
Process a dataset

Reads every file in the directory tree in parallel and writes a portable Parquet file containing per-file statistics, thumbnails, and metadata. Use -p to label subdirectories as separate conditions.

Step 2 pixel-patrol view out.parquet
Open the viewer

Starts a local HTTP server backed by native DuckDB and opens the viewer in your browser. Supports --group-by, --filter-col, --dim, and other flags to pre-configure the view.

pixel-patrol launch
Processing dashboard

Opens a Dash web UI for configuring processing parameters and monitoring progress in real-time. Opens the viewer automatically once processing completes.

pixel-patrol build-viewer-html -o out.html
Build static viewer

Exports the viewer as a self-contained HTML file or a GitHub Pages-style folder. Use this to host the viewer on a static site without a running Python process.

Quick example

# 1. Process a folder with two experimental conditions:
pixel-patrol process examples/datasets/bioio -o project.parquet \
  --loader bioio --name "my-project" \
  -p condition_a -p condition_b \
  -e tif -e nd2

# 2. Explore the result:
pixel-patrol view project.parquet

# 3. Open with a specific grouping pre-selected:
pixel-patrol view project.parquet --group-by condition --dim z=5 --dim t=0
📊

Statistical testing

The group comparison widget runs Mann-Whitney U tests with Bonferroni correction to flag significant differences between experimental conditions.

📤

Shareable reports

The Parquet file is the report. Anyone can open it in the hosted viewer, or you can export a fully self-contained HTML file for offline sharing.

🔌

Extensible by design

Add custom loaders, processors, and viewer widgets as Python packages. Entry points are discovered automatically — no fork or rebuild needed.

Extend Pixel Patrol

Pixel Patrol is designed to be extended. The examples/minimal-extension template shows how to register a custom loader, processor, and viewer widget as a standalone Python package. Entry points under pixel_patrol.loader_plugins, pixel_patrol.processor_plugins, and pixel_patrol.viewer_extensions are discovered automatically at runtime.
